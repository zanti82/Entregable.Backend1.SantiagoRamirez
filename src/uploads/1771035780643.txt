

### 1. Preparación y Configuración

Antes de empezar, Git necesita saber quién eres para marcar tus cambios.

* **`git config --global user.name "Tu Nombre"`**: Configura tu nombre de usuario.
* **`git config --global user.email "tu@email.com"`**: Configura tu correo electrónico.

---

### 2. El Ciclo de Trabajo Diario

Estos son los comandos que usarás mientras escribes código:

| Comando | Explicación |
| --- | --- |
| **`git status`** | **El más importante.** Te dice qué archivos has modificado y qué está listo para guardarse. Úsalo a cada rato. |
| **`git add .`** | Prepara **todos** los archivos modificados para ser guardados (los mete al "escenario" o *staging area*). |
| **`git add nombre-archivo`** | Igual que el anterior, pero solo prepara un archivo específico. |
| **`git commit -m "mensaje"`** | **Guarda la foto.** Crea un punto en la historia con un mensaje descriptivo de lo que hiciste. |

---

### 3. Conexión con GitHub (Remoto)

Para enviar y recibir el código de tus amigos:

* **`git remote -v`**: Te muestra a qué repositorio de GitHub está conectada tu carpeta local.
* **`git push origin nombre-rama`**: Sube tus *commits* (tus fotos guardadas) desde tu PC a GitHub.
* **`git pull origin nombre-rama`**: Baja los cambios que tus amigos subieron a GitHub y los mezcla con tu código local.
* **`git fetch`**: Solo pregunta "¿qué hay de nuevo?" en GitHub sin descargar ni mezclar nada todavía.

---

### 4. Manejo de Ramas (Branches)

Para trabajar en funciones nuevas sin romper el código principal:

* **`git branch`**: Lista todas las ramas locales.
* **`git checkout -b nombre-rama`**: Crea una rama nueva y te cambia a ella automáticamente.
* **`git checkout nombre-rama`**: Salta de una rama a otra ya existente.
* **`git merge nombre-rama`**: Estando en `main`, este comando trae los cambios de otra rama y los une al `main`.

---

### 5. ¿Cometiste un error? (Emergencias)

* **`git log`**: Muestra el historial de todos los cambios hechos (quién, cuándo y qué).
* **`git checkout -- nombre-archivo`**: Deshace los cambios que hiciste en un archivo y lo regresa a como estaba en el último *commit*.
* **`git reset --hard HEAD`**: **¡Cuidado!** Borra todos tus cambios actuales y te deja exactamente igual que el último *commit*.

---

### Resumen del flujo estándar:

Si acabas de sentarte a programar, el orden lógico suele ser:

1. `git pull` (Para tener lo último de tus amigos).
2. *Programas mucho...*
3. `git status` (Ves qué cambiaste).
4. `git add .` (Preparas todo).
5. `git commit -m "Arreglé el error del login"` (Guardas).
6. `git push origin mi-rama` (Lo subes a GitHub).
